O formulário HTML (`cadastro-produto.html`) envia os dados do produto por meio de uma requisição HTTP POST para o endpoint `/produtos/salvar`. No controlador (`ProdutoController`), os dados são automaticamente vinculados ao objeto `ProdutoModel` pelo mecanismo de data binding do Thymeleaf e submetidos à validação utilizando Bean Validation.
Caso sejam identificados erros de preenchimento ou violações das restrições definidas, o formulário é reapresentado ao usuário, exibindo mensagens de erro diretamente nos campos correspondentes por meio do componente `invalid-feedback`.
Além das validações estruturais, as regras de negócio são tratadas na camada de serviço (`ProdutoService`). Nessa etapa, são verificadas condições como a obrigatoriedade e não negatividade da quantidade em estoque, bem como a unicidade do nome do produto. Quando alguma regra é descumprida, o serviço lança exceções com mensagens descritivas, que são capturadas pelo controlador e convertidas em erros de validação apresentados ao usuário.
Antes da persistência dos dados, o serviço atualiza automaticamente o atributo `dataAtualizacao` com o valor atual de `LocalDateTime.now()`, registrando o momento exato da alteração. Após a operação ser concluída com sucesso, a aplicação exibe uma mensagem de confirmação contendo a data e o horário da atualização, permitindo ao usuário verificar quando o cadastro foi realizado ou modificado pela última vez.
